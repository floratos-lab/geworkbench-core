Project Folders Area

The Project Folders Area is located in the upper-left-hand corner of the application. This centralized workspace is used to manage projects, open data files and store intermediate analysis results during a caWorkbench session. When caWorkbench is launched, an empty Workspace folder is displayed. The folder can be populated by creating a new workspace or by loading a workspace that was saved previously.

Workspace

The workspace is capable of holding multiple projects, which are organized in a hierarchical or tree-like structure. The workspace can be renamed and saved in its entirety.  When an object in the Workspace, such as a project folder, contains objects, like datasets or images, it is preceded with an icon.

l           To view the next level down in the hierarchy, click on the “+” icon to expand the branch.

l           To collapse a branch, click on the “-” icon.

Datasets

Project folders appear as child nodes immediately under the workspace node. Each project may contain several heterologous datasets. These datasets can include input (source) data and derived data (results) associated with an experiment as well as image files. Source data can be loaded from the user’s local storage or from remote servers. The grouping of datasets in project folders within caWorkbench does not change their physical storage locations.
