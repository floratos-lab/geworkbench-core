Manage Projects

A project folder is used to hold a collection of related files and datasets. It allows you to organize files from several different locations into one location. Any number of projects can be created (as Project Folders) and managed in a single working session.  The application does not enforce project folder name uniqueness. If you wish, you can rename the newly created project node as well as delete a project.  If the project has been removed, any visualization panes displaying information related to the selected project will be updated (cleared) as well.

Create A Project:

1.          Right-click on the workspace folder icon (root element) or name and select New Project from the pop-up window.

2.          A new project node is created under the workspace root node with the default project name Project.

Alternatively, a new project node can be created using the main menu and selecting File->New->Project.

A Project Folder can be renamed from either the main menu or from within the Workspace. Renaming a project has no effect on the child nodes (datasets and images files) in the project folder.

Rename the Project Folder:

1.          Right click on the relevant project node

2.          Select Rename Project from the popup menu.

3.          Enter the new project name in the edit box and click OK.

Alternately, from the main menu select "Edit -> Rename -> Project" and enter the new project name.

Remove a Project:

1.          Right click on the relevant project node

2.          Select Remove Project from the popup menu.

3.          Enter the new project name in the edit box and click OK.

Upon the action, the workspace is updated with the selected project removed from its project tree. All visualization panes displaying information related to the selected project will be updated as well.
